# Xiaohongshu Content Capture

Use this skill to generate a daily traffic/content brief for a user-configured Xiaohongshu creator watchlist. The user-facing output is a Markdown document that must also be shown directly in the conversation.

## Watchlist Setup

This skill ships with no default creators. On first use, ask the user for the Xiaohongshu creators they want to watch, or let `scripts/collect_with_login.js` prompt for them interactively.

The watchlist lives in `config/creators.txt`, one creator per line.

To set or change the watchlist:

```bash
python3 scripts/update_creators.py --init
python3 scripts/update_creators.py --set @creatorA @creatorB
python3 scripts/update_creators.py --list
```

## Data Boundary

There is no official broad Xiaohongshu API assumed by this skill. Work only with:

- user-provided JSON/JSONL/CSV exports
- manually captured public post data
- the user's own authorized analytics exports
- compliant third-party data exports

Do not automate login, CAPTCHA bypass, anti-bot bypass, private API reverse engineering, or session-cookie reuse. If the user asks for direct platform scraping, ask for a compliant export or a manually captured dataset instead.

Read `references/compliance.md` when the user asks how to collect data.

## Expected Inputs

Prefer the login-assisted collection package generated by `scripts/collect_with_login.js`:

- `xhs-captures/xhs-watch-package-YYYY-MM-DD.json`

Legacy CSV/JSON/JSONL post exports remain supported, but do not surface CSV files to the user unless they explicitly ask for tabular data.

Read `references/data-schema.md` before mapping a new export.

## Visible Page Collection Helper

When the user wants to collect data from Xiaohongshu pages they can already view, use `helpers/visible_page_collector.js`.

The helper is a browser-side script/bookmarklet that:

- reads only the currently visible page DOM
- exports visible post records as JSON/CSV
- exports manually confirmed follower snapshots as JSON/CSV
- does not call Xiaohongshu APIs, automate login, reuse cookies, bypass CAPTCHA, or bypass anti-bot controls

Read `references/visible-page-collection.md` before helping the user install or run the helper.

## Login-Assisted Browser Collection

When the user explicitly authorizes use of their own logged-in Xiaohongshu account, use `scripts/collect_with_login.js`.

The workflow is:

1. If no watchlist exists, prompt the user for creator handles and save them to `config/creators.txt`.
2. Open a visible Chromium window with a local profile directory.
3. Let the user log in manually.
4. For each watched creator, automatically open Xiaohongshu search, click the best matching creator/profile result, and wait for the page to render.
5. Detect likely yesterday posts, open each post detail page, and extract visible title/body/metrics/source link.
6. If a visible video element is present, play it muted in the user's logged-in browser and save a few screenshot frames for AI-assisted visual review. Do not claim audio transcription unless captions or text are visible.
7. Save an internal JSON collection package.
8. Generate the Markdown brief with `scripts/daily_brief.py --package ...` and paste the report body directly in the chat.

Pause for user intervention only when automatic navigation cannot find the creator page or Xiaohongshu shows a verification/risk page.

Do not ask for the user's password, SMS code, cookies, or session token. Do not bypass CAPTCHA, device verification, risk checks, or rate limits.

Read `references/login-assisted-collection.md` before running this workflow.

## Quick Start

```bash
python3 scripts/daily_brief.py \
  --report-date 2026-07-06 \
  --package xhs-captures/xhs-watch-package-2026-07-06.json
```

Omit `--out` when the user wants the report shown in the conversation. If `--out` is provided, still paste the important report content in chat.

Date behavior:

- `--report-date` is the day the brief is generated.
- The report covers `report-date - 1 day`.
- If `--report-date` is omitted, the script uses the current date in Asia/Shanghai.

## Report Requirements

For each watched creator, include:

- creator account name
- follower count on the report day or latest available snapshot
- follower change versus the previous day
- yesterday's published post summaries based on detail-page text and, for videos, visible sampled frames/captions when available
- content analysis for each post and an overall analysis section
- likes, collects, comments for each yesterday post
- original Xiaohongshu link for each post

If a creator has no matching post yesterday, show that explicitly.

## Scripts

- `scripts/daily_brief.py`: generate the daily Markdown report; default output is stdout for direct chat display.
- `scripts/update_creators.py`: replace or print the watchlist.
- `scripts/collect_with_login.js`: open a visible browser for user-login-assisted creator collection, detail-page reading, and video-frame sampling.
- `helpers/visible_page_collector.js`: browser-side visible-page collection helper for manual exports.

Use `scripts/update_creators.py --list` to inspect the active watchlist.
